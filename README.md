# Commit-Narrator — 项目故事书

> 读 Git 历史，讲项目故事

分析任意 Git 仓库的提交历史，自动生成一份项目叙事——从第一行代码到最新提交，讲清楚这个项目是怎么演变的。

## 怎么用

在 Claude Code 里直接说就行：

```
帮我分析这个项目的 git 历史
```

```
谁贡献最多？哪些文件改动最频繁？
```

```
给新人写个项目介绍
```

```
分析一下 src/components 这个目录的历史
```

```
看看 2026 年以来的提交
```

```
分析一下张三的贡献
```

## 分析维度

| 维度 | 说明 |
|------|------|
| 项目概览 | 提交数、贡献者、时间跨度、代码增减量 |
| 提交分类 | 按新功能/修复/重构/文档/杂务自动分类统计 |
| 项目阶段 | 自动检测：项目启动 → 功能建设 → 稳定收敛 |
| 里程碑 | 初始化、大规模重构、版本发布、架构变更 |
| 高风险文件 | 最频繁变更 = 最容易出问题的文件 |
| 时间耦合 | 总是一起改动的文件，暴露架构耦合 |
| 巴士因子 | 知识集中度警告，单人掌控 = 高风险 |
| 贡献者画像 | 每人的代码量、专注领域、活跃天数 |
| 活跃度分析 | 每周/每天分布、编码高峰时段 |
| 开发节奏 | 加速/减速/稳定 |
| 新人阅读指南 | 从哪个提交开始读、看什么文件、跟谁的代码 |
